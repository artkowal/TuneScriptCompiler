# TuneScript Compiler - Command Line Interface (CLI)

This branch (`base-compiler-CLI`) contains the core command-line implementation of the TuneScript compiler. It is intentionally devoid of a Graphical User Interface (GUI) to focus strictly on the underlying compiler architecture, lexical analysis, and the code transformation pipeline. 

This project was developed as part of a Bachelor of Science (B.Sc.) degree in Computer Science, specializing in Software Engineering.

## Project Overview

**TuneScript** is a custom Domain-Specific Language (DSL) designed for algorithmic music composition. The compiler processes source code written in `.tune` files and performs a series of transformations to generate standard multi-track MIDI (`.mid`) files. 

The compilation process includes:
1. **Lexical and Syntactic Analysis**: Powered by ANTLR4.
2. **Abstract Syntax Tree (AST) Construction**: Building a structural representation of the source code.
3. **Semantic Analysis**: Verifying the logic, scopes, tempo, and time signatures.
4. **Intermediate Representation (IR) Generation**: Translating the validated AST into an internal representation.
5. **Code Generation (Backend)**: Encoding the IR instructions into a binary MIDI format format.

## Repository Structure

* `grammar/` - ANTLR4 grammar definitions for the TuneScript language (`.g4` extension).
* `generated/` - Python lexer, parser, and visitor components automatically generated by ANTLR4.
* `src/` - The core compiler engine, containing:
  * `ast_builder.py` / `ast_nodes.py` - AST node definitions and the tree construction logic.
  * `semantic_analyzer.py` - Semantic validation algorithms.
  * `ir_generator.py` - Transformation logic from AST to Intermediate Representation (IR).
  * `midi_generator.py` - The backend module utilizing the `mido` library to generate binary MIDI files.
* `examples/` - Sample TuneScript source code files.
* `output/` - The destination directory for the compiled `.mid` files.
* `main.py` - The primary entry point for the CLI application.

## Prerequisites and Installation

The software requires **Python 3.8** or higher.

1. **Clone the repository and switch to the CLI branch:**
```bash
git clone https://github.com/artkowal/TuneScriptCompiler.git
cd TuneScriptCompiler
git checkout base-compiler-CLI
```
   
2. Initialize a virtual environment (Recommended):

macOS/Linux:

```Bash
python3 -m venv .venv
source .venv/bin/activate
```
Windows:

```Bash
python -m venv .venv
.venv\Scripts\activate
```

3. Install dependencies:
Install the required libraries (including mido and the ANTLR4 runtime) via the provided requirements file:

```Bash
pip install -r requirements.txt
```
## Usage
To compile a TuneScript source file, execute the `main.py` script and pass the path to the `.tune` file as an argument.

Example command:

```Bash
python main.py examples/example.tune
```
Upon successful compilation, the standard output will display the execution logs of the subsequent compilation phases, and the resulting `.mid` file will be saved in the `output/` directory.
